# 專案規格書：DesktopCalendar (原生輕量桌面行事曆)

## 1. 專案概述
本專案為因應高資安需求環境（如公務機關或特定企業人員）所開發，旨在提供一個完全運行於本地端、高合規性且無須安裝的桌面議程工具。開發過程嚴格篩選技術堆棧，具備以下特點：
- **高安全性**：為符合內部安全政策，排除特定區域開發之第三方套件與服務。
- **純在地執行**：不具備任何網路連線功能，確保數據絕對隱私。
- **輕量免安裝**：採用原生技術，釘選於桌面底座，不干擾日常作業。
- **錯誤診斷**：內建 Logging 機制，自動記錄運作異常於執行檔目錄下的 `app.log`。

## 2. 核心技術指標
| 項目 | 規格 |
|---|---|
| 語言環境 | Python 3.8.10 (建議環境，相容 Windows 7) |
| GUI 框架 | Tkinter + colorchooser (色盤) |
| 影像處理 | Pillow (系統匣圖示) |
| 系統匣整合 | pystray |
| 系統介面 | Windows API (ctypes) |
| 部署方式 | 目錄形式 (`--onedir`) 執行檔，降低防毒誤判 |
| 資料儲存 | 本地 tasks.config，避免防毒誤判 |

## 3. 核心功能

### 3.1 視覺與顯示
- **可自訂底色**：透過設定介面的色盤自由選擇日曆底色。
- **透明度調整**：支援視窗透明度設定，讓日曆完美融入個人桌面背景。
- **永遠置底**：程式會自動釘選在系統桌面的最底層，不會妨礙或遮擋您日常操作的其他應用程式視窗。

### 3.2 互動操作
| 功能 | 操作說明 |
|---|---|
| **移動視窗** | 按住視窗任意位置拖拽 |
| **調整大小** | 拖拽右下角的 ⤡ 調整視窗寬高 |
| **新增/編輯記事** | 雙擊日期方格開啟記事視窗 |
| **開啟設定** | 點擊標題列右側 ⚙ 圖示 |

### 3.3 每日記事功能
- 雙擊任意日期開啟**可調整大小的記事編輯視窗**。
- **每行代表一個獨立事項**，支援多行輸入。
- **✔ 儲存**按鈕固定顯示於底部，或按 **Ctrl + Enter** 快速儲存。
- 日曆格內以項目符號（•）顯示前三項預覽，超出標示 `+(N項)`。
- 留空儲存（或清除全部後儲存）將刪除該日期的所有記事。

### 3.4 設定介面 (⚙ 圖示)
| 設定項目 | 說明 |
|---|---|
| **透明度** | 滑桿調整 30%~100%，**拖拽時即時預覽** |
| **啟動選項** | 勾選「開機時自動啟動」可將程式捷徑加入系統啟動資料夾 |
| **日曆底色** | 系統色盤選色，變更後**日曆所有元件同步更新** |
| **匯入行事曆** | 讀取外部設定檔，可選擇合併或取代 |
| **匯出行事曆** | 備份資料至外部設定檔 |
| **取消** | 還原所有設定至開啟前的狀態 |

### 3.5 系統整合
- **系統匣圖示**：右下角工作列僅顯示小圖示，不佔用工作列。
- **右鍵選單**：切換模式、開啟設定、結束程式。
- **位置與大小記憶**：程式關閉時自動記錄，下次啟動時回復。
- **錯誤紀錄**：若功能執行失敗，將自動寫入 `app.log` 檔案。

## 4. 快捷鍵總覽
| 快捷鍵 | 功能 |
|---|---|
| 雙擊日格 | 開啟當日記事 |
| `Ctrl + Enter` | 儲存記事（記事編輯框中）|

## 5. 資料結構 (tasks.config)
檔案採用 JSON 格式儲存，但副標名使用 `.config` 以降低防毒軟體 (如 Cortex XDR) 的誤判機率。

## 6. 打包規範 (Building)
執行 `build.bat` 即可一鍵打包：
```bash
python -m PyInstaller --onedir --noconsole --noupx --name="DesktopCalendar" main.py
```
> **依賴套件**：`pip install pyinstaller pillow pystray`

## 7. 排除項目 (Out of Scope)
- 排除特定敏感區域開發之第三方套件或函式庫，確保合規性。
- 不支援雲端同步，維持單機作業以確保資訊安全。
- 不使用 Electron、.NET 等重型框架。
- **作業系統相容性**：
    - 建議環境：Windows 10 或更高版本。
    - **Windows 7 支援**：雖然 Python 3.9+ 官方不支援 Windows 7，但本專案透過包含 `api-ms-win-core-path-l1-1-0.dll` 並修正路徑權限，仍可在 Windows 7 上正常運作。
